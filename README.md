# ZX_TechnologeTerm
知行科创——软件工程实训
软件工程实验项目

1. 用户端模块
定位：面向C端用户的全流程服务入口，支撑用户从“找店-选品-下单-售后”的核心操作。

核心功能：

用户身份管理：支持手机号/第三方（微信/支付宝）注册登录，维护用户基本信息（昵称、头像），账号安全（密码修改、绑定手机）。
地址与配送管理：用户可添加/编辑收货地址（含经纬度定位），系统自动计算配送范围与费用（如起送价、配送费、距离计费）。
个性化推荐：基于用户历史订单、收藏、搜索行为，通过算法推荐常购店铺、热门菜品或优惠活动。
消息触达：推送订单状态变更（接单/配送/完成）、优惠券到期提醒、平台活动通知等。
功能补充
支持用户收藏心仪店铺与菜品，下单时可快速选择；支持订单评价与晒图，为其他消费者提供参考，同时可对不良体验进行投诉反馈

2. 商家管理模块
定位：为入驻商家提供“商品-订单-数据”的一站式运营工具，提升经营效率。

核心功能：

商品与菜单管理：支持商家创建/编辑菜品（分类、价格、图片、描述），设置规格（如大小份、辣度），管理上下架状态及库存预警。
订单处理：实时接收用户订单，支持接单/拒单（需说明原因），同步打印小票；处理退单/改单请求（如用户取消、漏菜补发）。
营销工具：商家可自主配置满减活动（如“满30减5”）、折扣菜品、会员专属价，或参与平台级大促（如周末特惠）。
经营数据看板：统计当日/周/月订单量、销售额、热销菜品TOP10、用户复购率等，辅助优化选品与定价策略。
功能补充
支持商家绑定接单提醒设备，新订单实时弹窗提醒；可设置店铺营业状态与配送时段，适配不同经营场景

3. 订单履约模块
定位：串联用户、商家、骑手三方，保障订单从生成到完成的全链路闭环。

核心功能：

智能派单：判断用户位置
状态追踪：实时更新订单状态（待支付→已接单→制作中→配送中→已完成），用户可通过地图查看骑手位置。
异常处理：监控超时风险（如商家出餐慢、骑手配送延迟），触发预警并协调资源（如联系商家催单、更换骑手）。
结算对账：自动计算平台抽成、商家收入、骑手佣金，生成每日/每月结算账单，支持商家与骑手提现。
功能补充
支持用户主动联系骑手与商家，遇到配送异常可发起催单、取消订单申请，平台客服可介入处理异常订单

4. 营销与会员模块
定位：通过激励策略提升用户活跃度与商家粘性，驱动平台GMV增长。

核心功能：

优惠券体系：平台发放通用券（如新人券、满减券）、商家定向券（如关注店铺领券）；支持券的领取、核销、过期提醒。
会员成长体系：用户消费累积积分/等级（如普通→白银→黄金），对应享折扣、优先配送、专属客服等权益。
裂变拉新：设计邀请奖励（如老用户邀请新用户得红包）、拼团活动（多人拼单享低价），扩大用户规模。
精准营销：基于用户标签（如高频用户、高客单价用户）推送个性化活动（如生日券、复购券）。
功能补充
平台定期推出节日限定活动，结合会员等级发放专属福利，同时支持消费积分兑换精美周边礼品

5. 平台管理模块
定位：支撑平台方对全局业务、用户、商家的监管与运营决策。

核心功能：

权限与角色管理：划分超级管理员、区域运营、客服、财务等角色，控制各角色功能访问范围（如仅财务可查看结算数据）。
商家审核与风控：审核商家资质（营业执照、食品经营许可证），监控异常行为（如刷单、虚假交易），实施处罚（限流、封店）。
数据驾驶舱：汇总全平台核心指标（日活用户、订单量、GMV、客单价、骑手人效），支持多维度钻取分析（如按城市/时段/品类）。
系统配置：管理基础参数（如配送费规则、抽成比例、活动预算），维护字典表（如地区编码、菜品分类），保障系统灵活扩展。
功能补充
平台定期推出节日限定活动，结合会员等级发放专属福利，同时支持消费积分兑换精美周边礼品
